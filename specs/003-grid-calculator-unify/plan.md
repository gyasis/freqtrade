# Implementation Plan: Grid Calculator Unification & Module Portability

**Branch**: `003-grid-calculator-unify` | **Date**: 2026-03-22 | **Spec**: [spec.md](./spec.md)

---

## Summary

Eliminate the V1/V2 grid calculator split entirely. `grid_calculator.py` (V1) and `grid_calculator_v2.py` (V2) are both deleted and replaced by a single unified `GridCalculator`. `GridConfigV2` is renamed `GridConfig` and relocated to `config/grid_config.py` (shared contract, no calculator imports). Grid initialization moves from `adjust_position()` to `generate_entry_signal()` (where candle data is available) to unblock log-scale and auto-adjust features. `GridState.last_price` replaces the in-memory `_last_prices` dict, fixing the restart persistence gap. All external consumers update their import paths and method names.

## Implementation Phases

### Phase A 
 Config Layer (no breaking changes yet)

**Goal**: Create `GridConfig` in `config/grid_config.py`. Nothing else changes yet 
 both old files still exist. This phase is safe to review/test in isolation.

**Steps**:
1. Create `config/grid_config.py`:
   - Copy `GridConfigV2` fields verbatim 
 rename class to `GridConfig`
   - Move `VALID_METHODS`, `VALID_ALLOCATION_STRATEGIES`, `VALID_INDICATORS` constants here
   - Add `validate()` method (consolidates `GridCalculator.validate_grid` + `__post_init__` degenerate checks)
   - Ensure zero imports of pandas/numpy/talib (pure dataclass + stdlib only)
2. Add `GridConfig` to `config/__init__.py` exports
3. Write unit test for `GridConfig.validate()` edge cases

**Deliverable**: `config/grid_config.py` importable standalone. `GridConfigV2` in `grid_calculator_v2.py` still exists (not deleted yet).

---

### Phase B 
 Unified GridCalculator

**Goal**: Rewrite `grid_calculator.py` to absorb all V2 functions. `grid_calculator_v2.py` still exists but is no longer referenced by the module.

**Steps**:
1. Rewrite `modules/grid_trading/grid_calculator.py`:
   - Import `GridConfig` from `config.grid_config` (not from V2)
   - Implement `generate_levels(config, midprice, df=None)` 
 routes to linear or log path via config flags
   - Implement `calculate_midprice(df, config)` 
 routes to market_price / MA / mid_high_low / vwap
   - Rename all `get_*` methods: `levels_crossed_down`, `levels_crossed_up`, `nearest_level`, `level_below`, `level_above`
   - Absorb `calculate_allocation()` dispatcher from V2
   - Absorb `_auto_set_grid_parameters()` and `_auto_set_period()` as private helpers
   - Maintain talib optional guard (`try/except ImportError`)
2. Verify: no import from `grid_calculator_v2.py`
3. Update `modules/grid_trading/__init__.py` exports

**Deliverable**: Unified `GridCalculator` with full V2 feature set. Old V2 file not yet deleted.

### Phase C 
 GridState Update

**Goal**: Add `last_price` field, convert `grid_count` to property, fix `is_active()` semantics.

**Steps**:
1. Update `grid_state.py`:
   - Add `last_price: Optional[float] = None` field
   - Add `update_last_price(price: float) 
 None` method (sets `last_price` + `last_updated`)
   - Convert `grid_count` from stored field to `@property` returning `len(self.grid_levels)`
   - Remove `grid_count` from `__post_init__` constructor parameter and validation
   - Remove `len(grid_levels) != grid_count` check from `__post_init__` (now impossible to violate)
   - Update `is_active()`: return `self.last_price is not None`
   - Update `to_dict()`: add `"last_price"` key
   - Update `from_dict()`: read `"last_price"`, default to `initial_entry_price` if absent (legacy migration)
2. Update unit tests in `test_grid_module.py` for new semantics

**Deliverable**: `GridState` with `last_price` persistence. Both old calculator files still exist.

**Goal**: Move grid initialization to `generate_entry_signal()`, remove `_last_prices`, rename `_v2_configs`

**Steps**:

   - Update `on_order_filled()`: no change to logic, remove `_last_prices` reference if any
